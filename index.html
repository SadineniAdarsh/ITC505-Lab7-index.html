Mad Libs Game







Your Mad Lib Story:

This Mad Libs game is an interactive project that demonstrates the use of HTML, CSS, and JavaScript to create an engaging user experience. The game allows users to input various words (nouns, verbs, adjectives, adverbs, celebrities, and places) to generate a unique story.

HTML Tags Used:

  • DOCTYPE: Specifies the document type and version of HTML.
  • html: Defines the root element of an HTML document.
  • head: Contains meta-information about the document, such as the title and linked resources.
  • title: Sets the title of the document, visible in the browser tab.
  • style: Contains CSS rules that style the document's elements.
  • body: Contains the visible content of the document.
  • div: Defines a division or section in the document.
  • form: Encapsulates input fields and submission elements for user input.
  • input: Allows users to enter data and select options in the form.
  • datalist: Provides a list of predefined options for an input element.
  • button: Creates a clickable button for user interaction.
  • p: Defines a paragraph of text.
  • footer: Defines the footer section at the bottom of the document.
  • script: Embeds JavaScript code for interactivity and functionality.

CSS Styles Used:

  • body: Sets background, padding, font, and color properties for the body element.
  • .content-wrapper: Styles the div containing the form and story, adding padding, border-radius, and opacity.
  • input: Sets styling properties for input fields, including padding, width, and border-radius.
  • button: Specifies padding, margin, and background color for buttons.
  • #story: Styles the paragraph that displays the story, including padding, color, background color, and animation.
  • @keyframes: Defines the animation for fading in the story element.

JavaScript Functionality:

  • generateStory function: Constructs the Mad Libs story using user inputs.
  • Retrieval of user inputs: Gets the values from input fields for the story.
  • Story construction: Combines the user inputs to create a story.
  • Display of the story: Displays the generated story in the #story paragraph element.
  • Last modified date: Retrieves and displays the last modified date of the document in the footer.
